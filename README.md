# TextSummarization
Streamlit app for summarizing articles

## 🎯 О проекте

Сервис для автоматического создания краткого содержания длинных текстовых документов. Поддерживает форматы PDF, DOCX и TXT, обрабатывает документы до 100,000 символов с помощью Map-Reduce подхода и структурного чанкинга.

Проект демонстрирует полный ML-пайплайн: от парсинга документов и предобработки текста до инференса модели на GPU и веб-интерфейса.

## ✨ Возможности

- 📥 **Загрузка документов** в форматах PDF, DOCX, TXT
- 🧠 **Абстрактивная суммаризация** с помощью модели `rut5-base-absum` (T5-base, ~250M параметров)
-  **Структурный чанкинг** — разбиение по абзацам с сохранением смысловых границ
- 🔄 **Map-Reduce подход** для обработки длинных документов
-  **Веб-интерфейс** на Streamlit с отображением статистики и прогресса
- ⚡ **GPU-ускорение** через CUDA для быстрого инференса
- 📝 **Скачивание результата** в текстовом формате

## ️ Архитектура

### Пайплайн обработки

```
Документ (PDF/DOCX/TXT)
    ↓
Парсинг и извлечение текста
    ↓
Дедупликация абзацев
    ↓
Структурный чанкинг (по абзацам, max 400 токенов)
    ↓
Map: суммаризация каждого чанка моделью T5
    ↓
Объединение промежуточных саммари
    ↓
Reduce: финальная суммаризация (если нужно)
    ↓
Результат
```

### Ключевые компоненты

**`app/core/`** — ядро системы
- `model.py` — обёртка над HuggingFace Transformers для инференса
- `config.py` — настройки модели и параметров генерации

**`app/services/`** — сервисный слой
- `document_parser.py` — парсинг PDF (pdfplumber), DOCX (python-docx), TXT
- `chunking.py` — структурный чанкинг и Map-Reduce логика

**`app_streamlit.py`** — веб-интерфейс на Streamlit

## 🔧 Инженерные решения

### Структурный чанкинг

Вместо наивного разбиения текста каждые N токенов, реализован **семантический чанкинг по абзацам**:

- Текст разбивается на абзацы (минимальные смысловые единицы)
- Абзацы группируются в чанки до достижения лимита токенов
- Гигантские абзацы (>400 токенов) разбиваются жёстко с перекрытием 50 токенов
- **Результат**: модель получает законченные смысловые блоки, что снижает смешение фактов из разных разделов

### Дедупликация текста

При парсинге веб-страниц часто встречаются дубликаты абзацев (лид повторяется в заголовке и теле статьи). Реализована автоматическая очистка от точных дубликатов перед суммаризацией.

### Обработка сканов

Для PDF-файлов добавлена эвристика определения сканов: если извлечённый текст короче 100 символов, документ классифицируется как изображение и пользователю предлагается загрузить текстовый PDF.

### Ограничение размера

Документы ограничены 100,000 символов (~25,000 слов) для предотвращения чрезмерного времени обработки. При превышении лимита обрабатывается только первая часть с предупреждением пользователя.

## ⚙️ Параметры модели

Модель `rut5-base-absum` обучена на новостном датасете Gazeta и оптимизирована для абстрактивной суммаризации коротких текстов.

**Параметры генерации:**
- `max_input_tokens`: 450 (контекстное окно модели)
- `max_output_tokens`: 300 (длина саммари)
- `num_beams`: 4 (beam search для качества)
- `repetition_penalty`: 2.5 (штраф за повторы)
- `no_repeat_ngram_size`: 3 (запрет повторных n-грамм)
- `length_penalty`: 1.0 (нейтральное поощрение длины)

## 📊 Ограничения модели

Модель `rut5-base-absum` демонстрирует **высокое качество на коротких новостных текстах** (до 1000 слов), где она способна извлекать ключевые факты и генерировать связные саммари.

**Архитектурные ограничения Seq2Seq-подхода:**

- **Контекстное окно**: T5-base ограничен ~512 токенами, что требует чанкинга для длинных документов
- **Потеря глобального контекста**: при Map-Reduce подходе модель не видит весь документ целиком, что может приводить к смешению фактов из разных разделов
- **Стиль обучения**: модель обучалась на новостных лидах, поэтому на структурированных лонгридах (с подзаголовками, списками) может скатываться в экстрактивную суммаризацию (копирование ключевых фраз)
- **Отсутствие cross-chunk reasoning**: модель не может устанавливать связи между фактами из разных чанков

Эти ограничения являются фундаментальными для Seq2Seq-архитектуры и мотивируют переход к LLM-based подходам (RAG, prompt engineering) для сложных use-cases.

## 🚀 Установка и запуск

### Требования

- Python 3.10+
- CUDA-совместимый GPU (рекомендуется, но не обязателен)
- Windows/Linux/macOS

### Установка

```bash
# Клонировать репозиторий
git clone <your-repo-url>
cd MyTextSummarizer

# Создать виртуальное окружение
python -m venv venv

# Активировать (Windows)
venv\Scripts\activate
# Активировать (Linux/Mac)
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
```

### Запуск

```bash
streamlit run app_streamlit.py
```

Приложение откроется в браузере по адресу `http://localhost:8501`.

## 🧪 Использование

1. **Загрузите документ** (PDF, DOCX или TXT) через интерфейс Streamlit
2. **Нажмите "Суммаризировать"**
3. **Получите результат** — краткое содержание с возможностью скачивания

**Рекомендации:**
- Короткие новости (1-5 страниц) — оптимальное качество
- Научные статьи и отчёты (5-30 страниц) — хорошее качество
- Книги и романы — не рекомендуются (превышение лимита)
